Estimar historias de usuario

Gestión de Software 27 de abril, 2025

La estimación de historias de usuario es un paso esencial para asegurar una planificación efectiva en el marco de trabajo Scrum. Permite al equipo de desarrollo identificar el esfuerzo necesario para cada funcionalidad, facilitando la organización y gestión del backlog.

Historias de usuario

Historia de usuario 1: Como usuario, quiero ingresar el número de expediente en un formulario para poder buscar un trámite archivado específico.

Historia de usuario 2: Como usuario, quiero visualizar los resultados de búsqueda en una tabla paginada para leerlos con mayor facilidad.

Historia de usuario 3: Como usuario, quiero usar el mismo formulario de búsqueda para consultar trámites derivados sin tener que llenar datos adicionales.

Historia de usuario 4: Como sistema, quiero consultar al backend los trámites derivados para asegurar que los resultados mostrados sean precisos.

Historia de usuario 5: Como usuario, quiero ver los trámites derivados en una tabla paginada para revisarlos de manera ordenada.

Historia de usuario 6: Como usuario, quiero que toda la información del seguimiento de trámites se presente de forma clara para poder entender fácilmente el proceso.

Historia de usuario 7: Como usuario, quiero acceder a una tabla que registre cada paso del trámite para conocer su historial completo.

Historia de usuario 8: Como usuario, quiero ver los datos del trámite (código, tipo, asunto, fecha de registro e historial) para tener una visión detallada del proceso.

Historia de usuario 9: Como usuario, quiero contar con un buscador que filtre remitentes por nombre, celular e ID para encontrar rápidamente a la persona que necesito.

Historia de usuario 10: Como usuario, quiero recibir un mensaje cuando haya campos vacíos para evitar errores al buscar o editar remitentes.

Historia de usuario 11: Como usuario, quiero poder editar los datos de un administrador directamente desde la lista para actualizar la información rápidamente.

Historia de usuario 12: Como usuario, quiero ver una alerta de confirmación antes de eliminar un administrador para evitar borrados accidentales.

Herramientas

3.1 Métodos comunes de estimación

Para estimar las historias de usuario, el equipo Scrum utilizará el método Planning Poker. Este método facilita la discusión y el consenso sobre el esfuerzo requerido para cada historia, permitiendo asignar puntos de manera colaborativa.

Salidas

4.1 Historias de Usuarios Estimadas

Nuestro equipo de desarrollo estima el esfuerzo de cada historia de usuario en puntos de historia, lo cual esta estimaciones permiten una mejor planificación en los sprints y la entrega continua de valor al cliente.

Historia de usuario Valor final
Historia de usuario 1: Como usuario del sistema, quiero poder buscar un trámite archivado ingresando su número de expediente para visualizar su información detallada de manera rápida y organizada. 1
Historia de usuario 2: Como usuario del sistema, quiero consultar trámites derivados utilizando el mismo formulario de búsqueda para gestionar eficazmente los diferentes tipos de trámites desde un solo lugar. 3
Historia de usuario 3: Como usuario del sistema, quiero visualizar el seguimiento completo de un trámite con todos sus pasos registrados para entender su evolución y estado actual de manera clara. 8
Historia de usuario 4: Como usuario administrativo, quiero buscar, editar y eliminar remitentes desde un listado con filtros personalizados para mantener actualizada la base de datos de contactos. 5
Historia de usuario 5: Como usuario administrador, quiero gestionar la lista de administradores con funciones de edición y eliminación para mantener el control y la seguridad del sistema en tiempo real. 5
