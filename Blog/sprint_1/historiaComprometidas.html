Comprometer historias de usuario

Gestión de Software 27 de abril, 2025

El comprometer historias de usuario en Scrum es esencial porque asegura que el equipo trabaje de manera organizada y alineada con los objetivos del negocio, al establecer expectativas claras sobre lo que se debe entregar durante el sprint. Este compromiso fomenta la responsabilidad y la rendición de cuentas dentro del equipo, lo que facilita una planificación realista basada en la capacidad del equipo y evita sobrecargas.

Historias de Usuarios Estimadas

Nuestro equipo de desarrollo estima el esfuerzo de cada historia de usuario en puntos de historia, lo cual esta estimaciones permiten una mejor planificación en los sprints y la entrega continua de valor al cliente.

Historia de usuario Valor final
Historia de usuario 1: Como usuario del sistema, quiero poder buscar un trámite archivado ingresando su número de expediente para visualizar su información detallada de manera rápida y organizada. 1
Historia de usuario 2: Como usuario del sistema, quiero consultar trámites derivados utilizando el mismo formulario de búsqueda para gestionar eficazmente los diferentes tipos de trámites desde un solo lugar. 3
Historia de usuario 3: Como usuario del sistema, quiero visualizar el seguimiento completo de un trámite con todos sus pasos registrados para entender su evolución y estado actual de manera clara. 8
Historia de usuario 4: Como usuario administrativo, quiero buscar, editar y eliminar remitentes desde un listado con filtros personalizados para mantener actualizada la base de datos de contactos. 5
Historia de usuario 5: Como usuario administrador, quiero gestionar la lista de administradores con funciones de edición y eliminación para mantener el control y la seguridad del sistema en tiempo real. 5

Criterios para seleccionar historias comprometidas

Prioridad funcional

Se eligieron historias que abordan funcionalidades esenciales del sistema de trámites documentarios, como la búsqueda y seguimiento de trámites, por ser el núcleo operativo del sistema.

Valor para el usuario

Se priorizaron las historias que permiten una mejor experiencia de uso y mayor eficiencia para los trabajadores administrativos de TACIDH, como la edición de remitentes y visualización del historial de trámites.

Dependencias técnicas

Se eligieron historias que no dependían de otras funcionalidades aún no desarrolladas, lo cual permitió avanzar con autonomía en su implementación dentro del Sprint.

Viabilidad en el tiempo del Sprint

Se seleccionaron historias que podían completarse con los recursos y el tiempo disponible, basándonos en las estimaciones realizadas en las reuniones de planificación.

Salidas

Historias de usuario comprometidas

Historia de usuario 1: Como usuario, tener la opción de restablecer todos los campos del formulario de búsqueda con un solo clic, para agilizar la limpieza y reinicio de los filtros.

Historia de usuario 2: Como usuario del sistema, quiero poder localizar trámites archivados ingresando el número de expediente, para encontrar de manera rápida documentos específicos.

Historia de usuario 3: Como usuario, quiero realizar búsquedas de trámites derivados utilizando los mismos criterios que en los trámites archivados, con el fin de mantener una experiencia de uso uniforme y eficiente.

Historia de usuario 4: Como usuario, necesito buscar remitentes utilizando su nombre, número de celular o código de identificación, para acceder rápidamente a su información.

Historia de usuario 5: El usuario podrá buscar remitentes por nombre, celular o ID para ubicar rápidamente su información.
