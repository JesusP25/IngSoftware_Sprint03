10. Identificar Tareas

Ingeniería de Software 14 de Junio al 05 de Julio del 2025

La identificación de tareas es importante porque al momento de que el equipo se compromete a completar ciertas historias durante un sprint, es necesario dividirlas en tareas más pequeñas para facilitar la planificación, la asignación de responsabilidades y el seguimiento del progreso. Esta identificación permite al equipo trabajar de manera organizada y eficiente, asegurando que todos los miembros sepan exactamente qué deben hacer para alcanzar los objetivos del sprint y entregar valor al cliente dentro del tiempo establecido.

10.2. Historias de usuario comprometidas

Historia 01: Personalización de la interfaz institucional

Como usuario, quiero visualizar la plataforma con los colores institucionales, el logo y un diseño coherente para una experiencia alineada a la identidad de TACIDH.

Historia 2: Sección de consultas para usuarios remitentes

Como remitente, quiero teneruna sección de consultas para comunicarme con el administrador, de modo que pueda hacer seguimiento o resolver dudas.

Historia 3: Implementación de medidas de seguridad del sistema

Como administrador del sistema, quiero que se implementen medidas de seguridad para proteger el sistema de accesos no autorizados y vulnerabilidades.

Historia 4: Migración del sistema al servidor institucional de TACIDH

Como desarrollador, quiero migrar el sistema al servidor de TACIDH para que funcione correctamente en el entorno institucional.

HERRAMIENTAS

10.4. Reuniones de planificación de Iteración

En esta reunión, el equipo de desarrollo junto con el cliente revisan las historias de usuario priorizadas para la siguiente iteración. El equipo selecciona las historias que se pueden implementar según su capacidad, las descompone en tareas técnicas concretas, asigna responsabilidades y estima el esfuerzo necesario.

SALIDAS

10.5. Lista de tareas

Historia 1: Como usuario, quiero visualizar la plataforma con los colores institucionales, el logo y un diseño coherente para una experiencia alineada a la identidad de TACIDH.
  • Tarea 1.1: Ajustar CSS con paleta oficial de colores TACIDH.
  • Tarea 1.2: Insertar logo en header/footer y favicon.
  • Tarea 1.3: Estilizar botones, formularios y tablas con identidad visual.
  • Tarea 1.4: Probar la responsividad del diseño con técnicas de CSS.
  • Tarea 1.5: Validar accesibilidad mínima (contraste, legibilidad).
Historia 2: Como remitente, quiero tener una sección de consultas para comunicarme con el administrador, de modo que pueda hacer seguimiento o resolver dudas.
  • Tarea 2.1: Crear tabla mensajes en base de datos.
  • Tarea 2.2: Implementar formulario de envío con validación.
  • Tarea 2.3: Mostrar mensajes tipo chat con orden cronológico.
  • Tarea 2.4: Usar AJAX o WebSocket para mostrar nuevos mensajes sin recargar.
  • Tarea 2.5: Añadir autenticación y autorización para acceso a la vista.
Historia 3: Como administrador del sistema, quiero que se implementen medidas de seguridad para proteger el sistema de accesos no autorizados y vulnerabilidades.
  • Tarea 3.1: Integrar reCAPTCHA o Captcha en el login principal.
  • Tarea 3.2: Configurar .htaccess o reglas del servidor para bloquear acceso a directorios no permitidos.
  • Tarea 3.3: Revisar rutas públicas y privadas (proteger controladores o rutas sensibles).
  • Tarea 3.4: Implementar gestión de roles.
  • Tarea 3.5: Verificar sesiones y tokens en cada acción sensible.
Historia 4: Como desarrollador, quiero migrar el sistema al servidor de TACIDH para que funcione correctamente en el entorno institucional.
  • Tarea 4.1: Configurar entorno de producción en el servidor TACIDH.
  • Tarea 4.2: Exportar base de datos desde desarrollo y restaurar en el servidor.
  • Tarea 4.3: Configurar variables de entorno (credenciales, puertos, rutas).
  • Tarea 4.4: Probar acceso al sistema desde IPs internas/externas autorizadas.
  • Tarea 4.5: Realizar pruebas funcionales completas post-migración.
