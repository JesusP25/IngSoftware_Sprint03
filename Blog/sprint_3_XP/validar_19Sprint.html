19.2. Entregables del Sprint

Historia de Usuario Entregables
Historia de Usuario 1: Visualización de trámites por resolver Como usuario, quiero visualizar la plataforma con los colores institucionales, el logo y un diseño coherente para una experiencia alineada a la identidad de TACIDH
1. Ajustar CSS con paleta oficial de colores TACIDH.
2. Insertar logo en header/footer y favicon.
3. Estilizar botones, formularios y tablas con identidad visual.
4. Probar responsividad del diseño con técnicas de CSS.
5. Validar accesibilidad mínima (contraste, legibilidad).
Historia de Usuario 2: Búsqueda de trámites por resolver Como remitente, quiero tener una sección de consultas para comunicarme con el administrador, de modo que pueda hacer seguimiento o resolver dudas.
1. Crear tabla mensajes en la base de datos.
2. Implementar formulario de envío con validación.
3. Mostrar mensajes tipo chat con orden cronológico.
4. Usar AJAX o WebSocket para mensajes en tiempo real
5. Añadir autenticación y autorización para acceso a la vista.
Historia de Usuario 3: Mensaje de trámites pendientes por resolver Como administrador del sistema, quiero que se implementen medidas de seguridad para proteger el sistema de accesos no autorizados y vulnerabilidades.
1. Integrar reCAPTCHA o Captcha en el login principal
2. Configurar .htaccess o reglas de servidor para proteger directorios
3. Revisar rutas públicas y privadas (proteger controladores o rutas sensibles).
4. Implementar gestión de roles.
5. Verificar sesiones y tokens en acciones sensibles.
Historia de Usuario 4: Visualización de trámites por recibir Como desarrollador, quiero migrar el sistema al servidor de TACIDH para que funcione correctamente en el entorno institucional.
1. Configurar el entorno de producción en el servidor TACIDH.
2. Exportar base de datos desde desarrollo y restaurar en el servidor
3. Configurar variables de entorno (credenciales, puertos, rutas)
4. Probar acceso desde IPs internas/externas autorizadas.
5. Realizar pruebas funcionales completas post-migración.

19.4. Criterios terminados

Funcionalidad completada:

  • La historia cumple todos los criterios de prueba de aceptación definidos previamente.

Pruebas realizadas

  • Se ejecutaron y aprobaron pruebas unitarias, de integración y de aceptación (si aplica).

Código revisado

  • Se revisó en pareja o por un compañero, y se refactorizó eliminando duplicaciones o partes innecesarias.

Diseño validado

  • La interfaz sigue la identidad visual de TACIDH y es funcional en escritorio y móvil.

Documentación actualizada

  • Se documentaron cambios en código, rutas, roles, entidades y archivos técnicos.

Integración completada

  • El código está en el repositorio y ha sido integrado correctamente en el entorno de despliegue.

Feedback del cliente validado

  • El cliente aprobó la funcionalidad mediante demo o prueba directa.

19.5. Criterios de Prueba de Aceptación de las historias del usuario

Historia Criterios de Prueba de aceptación
Historia 1 - El logo institucional es visible en todas las vistas.
- Se aplican los colores oficiales en menús, botones y encabezados.
- La tipografía es legible y estandarizada.
- El diseño responde correctamente en pantallas de PC y móvil.
Historia 2 - Existe un formulario para enviar consultas.
- El remitente puede visualizar respuestas del administrador.
- Las respuestas se reciben sin necesidad de recargar la página.
- Solo usuarios autenticados pueden enviar y ver mensajes.
Historia 3 - El sistema integra Captcha en formularios críticos.
- No existen directorios públicos accesibles vía navegador.
- Se gestionan permisos por roles: admin, remitente, revisor, etc.
- Se restringe el acceso a rutas según tipo de usuario.
Historia 4 - El sistema es accesible desde el dominio interno.
- La base de datos ha sido migrada sin pérdida de datos.
- Los enlaces y rutas funcionan en producción.
- El sistema mantiene su rendimiento y seguridad tras la migración.
