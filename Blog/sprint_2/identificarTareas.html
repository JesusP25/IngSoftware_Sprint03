10. Identificar tareas

Gestión de Software 17 / 05 al 07 / 06 - 2025

La identificación de tareas es importante porque al momento de que el equipo se compromete a completar ciertas historias durante un sprint, es necesario dividirlas en tareas más pequeñas para facilitar la planificación, la asignación de responsabilidades y el seguimiento del progreso. Esta identificación permite al equipo trabajar de manera organizada y eficiente, asegurando que todos los miembros sepan exactamente qué deben hacer para alcanzar los objetivos del sprint y entregar valor al cliente dentro del tiempo establecido.

10.2. Historias de usuario comprendidas

Historia de usuario 1: Como usuario, quiero ver en pantalla una lista de trámites por resolver con los campos: urgente, n.º expediente, hora, fecha, área de origen, estado y área de destino, para gestionar de manera organizada los trámites pendientes.

Historia de usuario 2: Como usuario, quiero contar con un buscador en la vista de trámites por resolver, para filtrar rápidamente el trámite que deseo revisar.

Historia de usuario 3: Como usuario, quiero ver un mensaje que me indique cuántos trámites tengo por resolver, para tener claridad sobre mi carga de trabajo.

Historia de usuario 4: Como usuario, quiero visualizar los trámites por recibir con todos los campos indicados, para gestionar las recepciones de documentos.

Historia de usuario 5: Como usuario, quiero tener un buscador funcional en la sección de trámites por recibir, para encontrar rápidamente un trámite específico.

Historia de usuario 6: Como usuario, quiero ver un mensaje con la cantidad de trámites que tengo por recibir, para mantenerme informado al momento.

Historia de usuario 7: Como usuario, quiero tener una opción para imprimir trámites de remitentes externos, para generar reportes físicos cuando se requiera.

Historia de usuario 8: Como usuario del sistema, quiero aplicar filtros por tipo de documento y fecha de recepción en los trámites remitentes externos, para facilitar la búsqueda de registros específicos y mejorar la gestión documental.

Historia de usuario 9: Como usuario, quiero registrar trámites internos con los campos requeridos, para documentar adecuadamente su origen y destino.

Historia de usuario 10: Como usuario remitente, quiero recibir alertas si dejo campos obligatorios vacíos al registrar un trámite interno, para asegurar que el registro sea correcto y completo.

Historia de usuario 11: Como administrador, quiero ver, buscar, editar y eliminar áreas, para gestionar eficientemente la estructura organizacional.

Historia de usuario 12: Como administrador, quiero navegar entre páginas en el listado de áreas, para revisar todas las áreas registradas sin sobrecargar la vista.

Historia de usuario 13: Como administrador, quiero buscar, editar y eliminar tipos de documentos, para mantener actualizada la clasificación documental.

Historia de usuario 14: Como administrador, quiero contar con una barra de búsqueda en la gestión de tipos de documentos, para encontrar rápidamente el documento que necesito editar o eliminar.

HERRAMIENTAS

10.3. Reuniones de planificación del Sprint

En esta reunión, el equipo Scrum, junto con el Product Owner, define qué historias de usuario o elementos del backlog se priorizan y se comprometen a completarlas dentro del sprint. El equipo también desglosa estas historias en tareas más pequeñas, asigna responsabilidades y estima el esfuerzo necesario.

SALIDAS

10.4. Lista de tareas

Como resultado de la planificación, se genera una lista detallada de tareas comprometidas, con responsables asignados y fechas estimadas para su entrega dentro del sprint.

Historia 1:

Como usuario, quiero ver en pantalla una lista de trámites por resolver con los campos: urgente, n.º expediente, hora, fecha, área de origen, estado y área de destino, para gestionar de manera organizada los trámites pendientes.

  • Tarea 1.1: Diseñar tabla con campos requeridos.
  • Tarea 1.2: Implementar lógica para mostrar datos paginados.
  • Tarea 1.3: Agregar indicador de página actual.

Historia 2:

Como usuario, quiero contar con un buscador en la vista de trámites por resolver, para filtrar rápidamente el trámite que deseo revisar.

  • Tarea 2.1: Implementar campo de búsqueda.

Historia 3:

Como usuario, quiero ver un mensaje que me indique cuántos trámites tengo por resolver, para tener claridad sobre mi carga de trabajo.

  • Tarea 3.1: Calcular trámites pendientes.
  • Tarea 3.2: Mostrar mensaje con el conteo.

Historia 4:

Como usuario, quiero visualizar los trámites por recibir con todos los campos indicados, para gestionar las recepciones de documentos.

  • Tarea 4.1: Diseñar tabla.
  • Tarea 4.2: Implementar paginación.

Historia 5:

Como usuario, quiero tener un buscador funcional en la sección de trámites por recibir, para encontrar rápidamente un trámite específico.

  • Tarea 5.1: Crear componente de búsqueda.
  • Tarea 5.2: Integrar con tabla de resultados.

Historia 6:

Como usuario, quiero ver un mensaje con la cantidad de trámites que tengo por recibir, para mantenerme informado al momento.

  • Tarea 6.1: Obtener la cantidad de trámites por recibir.
  • Tarea 6.2: Mostrar mensaje con esa cantidad.

Historia 7:

Como usuario, quiero tener una opción para imprimir trámites de remitentes externos, para generar reportes físicos cuando se requiera.

  • Tarea 7.1: Agregar botón "Imprimir".
  • Tarea 7.2: Generar plantilla de impresión.

Historia 8:

Como usuario del sistema, quiero aplicar filtros por tipo de documento y fecha de recepción en los trámites remitentes externos, para facilitar la búsqueda de registros específicos y mejorar la gestión documental.

  • Tarea 8.1: Implementar lógica de filtrado en backend por tipo y fecha.
  • Tarea 8.2: Mostrar mensaje si la búsqueda no arroja resultados.

Historia 9:

Como usuario, quiero registrar trámites internos con los campos requeridos, para documentar adecuadamente su origen y destino.

  • Tarea 9.1: Diseñar formulario con campos necesarios.
  • Tarea 9.2: Validar datos antes de enviar.
  • Tarea 9.3: Crear endpoint de registro de trámites.

Historia 10:

Como usuario remitente, quiero recibir alertas si dejo campos obligatorios vacíos al registrar un trámite interno, para asegurar que el registro sea correcto y completo.

  • Tarea 10.1: Definir campos obligatorios en el formulario.
  • Tarea 10.2: Implementar validaciones en frontend (con mensajes de error).
  • Tarea 10.3: Agregar validación también en backend para seguridad adicional.

Historia 11:

Como administrador, quiero ver, buscar, editar y eliminar áreas, para gestionar eficientemente la estructura organizacional.

  • Tarea 11.1: Diseñar tabla de áreas.
  • Tarea 11.2: Crear buscador funcional.
  • Tarea 11.3: Implementar botones de edición y eliminación.
  • Tarea 11.4: Agregar contador de total de áreas.
  • Tarea 11.5: Habilitar paginación.

Historia 12:

Como administrador, quiero navegar entre páginas en el listado de áreas, para revisar todas las áreas registradas sin sobrecargar la vista.

  • Tarea 12.1: Agregar componente de paginación en tabla de áreas.
  • Tarea 12.2: Implementar lógica backend de paginado.
  • Tarea 12.3: Mostrar cantidad total de páginas y resultados.

Historia 13:

Como administrador, quiero buscar, editar y eliminar tipos de documentos, para mantener actualizada la clasificación documental.

  • Tarea 13.1: Crear tabla de tipos documentales.
  • Tarea 13.2: Agregar botones de edición y eliminación.
  • Tarea 13.3: Implementar buscador por nombre.

Historia 14:

Como administrador, quiero contar con una barra de búsqueda en la gestión de tipos de documentos, para encontrar rápidamente el documento que necesito editar o eliminar.

  • Tarea 14.1: Crear campo de búsqueda en la interfaz.
  • Tarea 14.2: Adaptar el backend para realizar consultas por nombre/tipo.
  • Tarea 14.3: Manejar el caso de cero resultados con mensaje amigable.
