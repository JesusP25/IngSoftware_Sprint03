16.2. Entregables del Sprint

Historia de Usuarios Entregables
Historia de usuario 01: Visualización de trámites por resolver Como usuario remitente, quiero recibir alertas si dejo campos obligatorios vacíos al registrar un trámite interno, para asegurar que el registro sea correcto y completo.
1. Tabla con campos: Urgente, N° de expediente, Hora, Fecha, Área de origen, Estado, Área de destino.
2. Componente de paginación con número de página actual.
3. Visualización de datos en tabla paginada.
4. Datos obtenidos correctamente desde backend.
Historia de usuario 02: Búsqueda de trámites por resolver Como usuario remitente, quiero recibir alertas si dejo campos obligatorios vacíos al registrar un trámite interno, para asegurar que el registro sea correcto y completo.
1. Campo de búsqueda funcional.2. Resultados filtrados en tabla con base al número de expediente.
2. Resultados filtrados en tabla con base al número de expediente.
3.Comportamiento reactivo al escribir o hacer clic en “Buscar”.
Historia de usuario 03: Mensaje de trámites pendientes por resolver Como usuario, quiero ver un mensaje que me indique cuántos trámites tengo por resolver, para tener claridad sobre mi carga de trabajo.
1. Lógica para contar trámites pendientes desde backend.
2. Mensaje en UI: “Usted tiene X expedientes por resolver.”.
3. Estilo de mensaje visible y destacado en la interfaz.
Historia de usuario 04: Visualización de trámites por recibir Como usuario, quiero visualizar los trámites por recibir con todos los campos indicados, para gestionar las recepciones de documentos.
1. Tabla con campos: N° de expediente, Hora, Fecha, Área de origen, Estado, Área de destino.
2. Implementación de paginación.
3. Resultados correctamente cargados y ordenados.
Historia de usuario 05: Búsqueda de trámites por recibir Como usuario, quiero tener un buscador funcional en la sección de trámites por recibir, para encontrar rápidamente un trámite específico.
1. Campo de búsqueda funcional en interfaz.
2. Resultados filtrados dinámicamente.
3. Integración entre componente de búsqueda y tabla de datos.
Historia de usuario 06: Mensaje de trámites por recibir Como usuario, quiero ver un mensaje con la cantidad de trámites que tengo por recibir, para mantenerme informado al momento.
1. Backend calcula la cantidad de trámites pendientes por recibir.
2. Visualización en UI: “Usted tiene X expedientes por recibir.”
3. Validación de que el conteo es exacto y se actualiza.
Historia de usuario 07: Impresión de trámites externos Como usuario, quiero tener una opción para imprimir trámites de remitentes externos, para generar reportes físicos cuando se requiera.
1. Botón de impresión habilitado en interfaz.
2. Generación de vista o plantilla para impresión.
3. Pruebas de impresión con diferentes datos.
Historia de usuario 08: Filtro para trámites remitentes externos Como usuario del sistema, quiero aplicar filtros por tipo de documento y fecha de recepción en los trámites remitentes externos, para facilitar la búsqueda de registros específicos y mejorar la gestión documental.
1. Backend con lógica para filtrar por tipo y fecha.
2. Interfaz que muestra mensaje si no hay resultados.
3. Filtros funcionales y resultados precisos.
Historia de usuario 09: Registro de trámites internos Como usuario, quiero registrar trámites internos con los campos requeridos, para documentar adecuadamente su origen y destino.
1. Formulario de registro completo con campos necesarios.
2. Validaciones frontend activas.
3. Endpoint en backend para registrar el trámite.
Historia de usuario 10: Validación de campos para trámites interno Como usuario remitente, quiero recibir alertas si dejo campos obligatorios vacíos al registrar un trámite interno, para asegurar que el registro sea correcto y completo.
1. Validaciones en frontend con mensajes de error.
2. Validaciones adicionales en backend.
3. Campos obligatorios claramente definidos.
Historia de usuario 11: Administrar lista de áreas Como administrador, quiero ver, buscar, editar y eliminar áreas, para gestionar eficientemente la estructura organizacional.
1. Tabla de áreas con campos visibles.
2. Buscador funcional para filtrar por nombre.
3. Botones de edición y eliminación funcionando.
4. Contador de total de áreas visible.
Historia de usuario 12: Navegación en listado de áreas Como administrador, quiero navegar entre páginas en el listado de áreas, para revisar todas las áreas registradas sin sobrecargar la vista.
1. Lógica de paginado implementada en backend.
2. Interfaz con paginador y navegación.
3. Visualización de total de páginas y registros.
Historia de usuario 13: Administrar tipos de documentos Como administrador, quiero buscar, editar y eliminar tipos de documentos, para mantener actualizada la clasificación documental.
1. Tabla con listado de tipos documentales.
2. Botones de editar y eliminar visibles y funcionales.
3. Componentes estilizados para buena usabilidad.
Historia de usuario 14: Implementar búsqueda en tipos de documentos Como administrador, quiero contar con una barra de búsqueda en la gestión de tipos de documentos, para encontrar rápidamente el documento que necesito editar o eliminar.
1. Campo de búsqueda en tabla de tipos documentales.
2. Lógica backend para filtrar por nombre o tipo.
3. Mensaje visible si no se encuentran resultados.

16.4. Criterios terminados

  • Épica 1: Trámites por resolver
    • El sistema muestra una tabla con las columnas: Urgente (Sí/No), N° de expediente, hora, fecha, área de origen, estado y área de destino.
    • En la parte superior se visualiza el mensaje: “Usted tiene X expedientes por resolver”.
    • El formulario de búsqueda permite filtrar trámites por número de expediente u otros criterios.
    • Se visualiza la paginación correctamente con el número de página actual.
    • El usuario puede navegar entre páginas sin perder los filtros aplicados.
  • Épica 2: Trámites por recibir
    • La tabla contiene las columnas: N° de expediente, hora, fecha, área de origen, estado y área de destino.
    • Se muestra el mensaje “Usted tiene X expedientes por recibir” dinámicamente en la parte superior.
    • El buscador filtra correctamente los trámites según lo ingresado.
    • Se visualiza el número de página en que se encuentra el usuario.
    • Al marcar un trámite como recibido, este se actualiza correctamente en la base de datos.
  • Épica 3: Trámites remitentes externos
    • La vista permite imprimir el listado completo de trámites con los campos necesarios.
    • Los campos mostrados coinciden con los de la tabla de “Trámites por recibir”.
    • Se incluyen filtros por fecha, tipo de documento y N° de expediente.
    • El botón de impresión genera una vista previa limpia y ordenada.
    • Si no hay datos que coincidan con los filtros, se muestra un mensaje adecuado.
  • Épica 4: Trámites remitentes internos
    • El formulario contiene los campos: Fecha de registro, hora de registro, tipo de trámite, N° de expediente, siguiente unidad orgánica de destino, documento (múltiples archivos), asunto, folio, remitente y observaciones.
    • Todos los campos obligatorios deben completarse antes de poder guardar el trámite.
    • Se validan los tipos y tamaños de archivos en el campo de documentos.
    • Se muestra un mensaje de éxito una vez que el trámite ha sido registrado.
    • Las observaciones se registran correctamente y pueden ser visualizadas posteriormente.
  • Épica 5: Gestión de áreas
    • La tabla muestra correctamente todas las áreas registradas.
    • Las acciones de editar y eliminar están disponibles junto a cada área.
    • El buscador filtra resultados en tiempo real por nombre de área.
    • El sistema muestra un contador con la cantidad total de áreas.
    • Los botones “Siguiente” y “Anterior” funcionan correctamente en la navegación por páginas.
    • Se muestra una alerta de confirmación al intentar eliminar un área.
  • Épica 6: Gestión de tipo de documentos
    • La lista de documentos permite editar o eliminar cada tipo desde acciones específicas.
    • El buscador filtra correctamente por nombre del documento.
    • Solo usuarios con permisos adecuados pueden modificar o eliminar un tipo de documento.
    • Se muestran mensajes de confirmación al eliminar un tipo de documento.
    • Se guarda correctamente cualquier edición hecha en el nombre o descripción del documento.
    • Se informa al usuario si no se encuentran documentos con los criterios buscados.
